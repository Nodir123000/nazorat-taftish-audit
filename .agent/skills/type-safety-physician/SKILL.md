# Type-Safety Physician (Legendary Mode)

Этот скилл предназначен для автономного исправления ошибок TypeScript в проекте АИС КРР. Он сочетает в себе строгую проверку типов и интеллектуальное исправление «хвостов» после рефакторинга.

## 1. Режимы работы

### Gold Master (Технический)
- **Цель**: Полное отсутствие ошибок компиляции (`npm run build` проходит).
- **Действия**:
  - Исправление `implicit any` путем добавления явных типов или `: any`.
  - Исправление несовпадения типов в Props/State.
  - Удаление неиспользуемых импортов.

### Legendary (Семантический)
- **Цель**: Консистентность бизнес-логики и терминологии.
- **Действия**:
  - **Terminology Sync**: Автоматическая замена устаревших терминов (например, `Ревизор` -> `Член комиссии`, `mobilePhone` -> `contactPhone`).
  - **Predictive Healing**: Если меняется интерфейс в `lib/types/`, агент находит все места использования и обновляет доступ к свойствам.
  - **Boundary Guard**: Перенос серверной логики из клиентских компонентов в сервисы.

## 2. Глоссарий исправлений (Terminology Map)

| Устаревший термин | Новый стандарт (Legendary) | Область |
| :--- | :--- | :--- |
| `Главный ревизор` | `Председатель комиссии` | `CommissionRole` |
| `Ревизор` | `Член комиссии` | `CommissionRole` |
| `mobilePhone` | `contactPhone` | `PersonnelMember` |
| `personalPhone` | `contactPhone` (если нет `personalPhone`) | `Inspector` |
| `audit_id` | `planId` | `AuditRecord` |

## 3. Алгоритм «Legendary Fix»

1. **Scan**: Чтение вкладки "Problems" или запуск `npx tsc --noEmit`.
2. **Classify**: Группировка ошибок по файлам и типам.
3. **Execute**:
   - Если ошибка `Property X does not exist on type Y`: проверить глоссарий (пункт 2).
   - Если ошибка `Implicit any`: добавить `: any` и оставить комментарий `// TODO: Refine type`.
   - Если ошибка `Type mismatch`: привести к базовому интерфейсу или использовать `as any` в крайнем случае для прохождения билда.
4. **Verify**: Повторная проверка только исправленных файлов.

## 4. Критические правила

- **Никогда** не удаляй бизнес-логику для исправления типов.
- **Никогда** не оставляй `any` там, где можно легко вывести тип из `lib/types/`.
- Сохраняй комментарии и документацию кода.
