# Жизненный цикл ревизии в АИС КРР

## Схема статусов

```
[draft] → [planned] → [in_progress] → [completed] → [archived]
                           ↓
                      [suspended] → [in_progress]
```

## Описание каждого этапа

### 1. `draft` — Черновик
- **Кто создает**: `inspector`, `chief_inspector`
- **Что доступно**: Редактирование всех полей
- **Что нельзя**: Добавлять нарушения
- **Следующий шаг**: Создание приказа `orders` и перевод в `planned`

### 2. `planned` — Запланировано
- **Триггер**: Создан приказ (`orders.status = 'approved'`)
- **Что доступно**: Назначение членов комиссии (`commission_members`)
- **Обязательно**: Минимум 1 ответственный (`is_responsible = true`)
- **Следующий шаг**: Дата начала наступила → `in_progress`

### 3. `in_progress` — В работе
- **Что доступно**: Добавление нарушений (`violations`), заполнение актов
- **Статус нарушений**: Можно создавать в статусе `registered` или `draft`
- **Следующий шаг**: Все нарушения обработаны → `completed`

### 4. `suspended` — Приостановлено
- **Когда**: Форс-мажор, дополнительные материалы
- **Требует**: Основание для приостановки (текстовое поле)
- **Возврат**: Всегда возвращается в `in_progress`

### 5. `completed` — Завершено
- **Что доступно**: Только просмотр, экспорт отчетов
- **Обязательно**: Акт ревизии (`audit_reports`) должен быть в статусе `final`
- **Нельзя**: Добавлять новые нарушения

### 6. `archived` — Архив
- **Когда**: Через 90 дней после `completed` или вручную администратором
- **Что доступно**: Только чтение

---

## Документы на каждом этапе

| Этап | Документ | Таблица БД |
| :--- | :--- | :--- |
| planned | Приказ о проведении ревизии | `orders` |
| in_progress | Рабочий материал, акт проверки | `findings` |
| in_progress | Акт выявленных нарушений | `violations` |
| completed | Итоговый акт ревизии | `audit_reports` |
| completed | Решения по нарушениям | `decisions` |

## Роли и доступ по этапам

| Роль | draft | planned | in_progress | completed |
| :--- | :--- | :--- | :--- | :--- |
| `admin` | R/W | R/W | R/W | R |
| `chief_inspector` | R/W | R/W | R/W | R |
| `inspector` | R/W (свои) | R | R/W (свои) | R |
| `viewer` | — | R | R | R |
