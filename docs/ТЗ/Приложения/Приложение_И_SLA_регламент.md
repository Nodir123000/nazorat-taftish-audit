## И.1. ОБЩИЕ ПОЛОЖЕНИЯ

### И.1.1. Назначение

Настоящее приложение определяет регламент технической поддержки АИС КРУ МО РУЗ, включая каталог сервисов, уровни сервиса (SLA), процедуры эскалации и контакты поддержки.

### И.1.2. Термины и определения

| Термин | Определение |
|--------|-----------|
| **Инцидент** | Любое событие, нарушающее нормальное функционирование системы |
| **Запрос на обслуживание** | Обращение пользователя с просьбой о помощи или предоставлении информации |
| **SLA** | Service Level Agreement — соглашение об уровне сервиса |
| **MTTR** | Mean Time To Repair — среднее время устранения |
| **Эскалация** | Передача инцидента на более высокий уровень |

### И.3.1. Определение уровней

| Уровень | Доступность | Время простоя/мес | Примеры |
|---------|------------|-------------------|--------|

| **Bronze** | 99.5% | ≤ 3.7 ч | Справочники, Help, Архив, Мониторинг |

| **MTTR (высокий)** | ≤ 2 часа |
| **Плановый downtime** | ≤ 30 мин/месяц |
| **Уведомление о downtime** | За 24 часа |

### И.3.3. SLA для Silver-сервисов

| Метрика | Значение |
|---------|---------|
| **Доступность** | 99.9% |
| **Время отклика (p95)** | ≤ 3 сек |
| **MTTR (высокий)** | ≤ 4 часа |
| **MTTR (средний)** | ≤ 8 часов |
| **Плановый downtime** | ≤ 2 ч/месяц |

### И.3.4. SLA для Bronze-сервисов

| Метрика | Значение |
|---------|---------|

| **Плановый downtime** | ≤ 4 ч/месяц |

## И.4. УРОВНИ ИНЦИДЕНТОВ

### И.4.1. Классификация инцидентов

| Уровень | Название | Влияние | Время реакции | Время решения |
|---------|---------|---------|--------------|---------------|
| **P1** | Критический | Система недоступна для всех пользователей | 15 мин | 1 час |
| **P2** | Высокий | Основная функция недоступна | 30 мин | 4 часа |
| **P3** | Средний | Функция работает частично | 2 часа | 8 часов |
| **P4** | Низкий | Незначительное влияние | 4 часа | 24 часа |
| **P5** | Косметический | UI-issues, некритичные | 8 часов | 48 часов |

### И.4.2. Описание уровней

#### P1 — Критический

| Параметр | Описание |
|----------|---------|
| **Определение** | Полная недоступность системы или критической функции |
| **Примеры** | Не работает вход в систему, падение базы данных |
| **Реакция** | Немедленная, 24/7 |
| **Эскалация** | Руководитель проекта, CTO |

#### P2 — Высокий

| Параметр | Описание |
|----------|---------|
| **Определение** | Критическая функция недоступна для группы пользователей |
| **Примеры** | Не создаются ревизии, не формируются акты |
| **Реакция** | В рабочее время, 30 мин |
| **Эскалация** | Менеджер проекта |

#### P3 — Средний

| Параметр | Описание |
|----------|---------|
| **Определение** | Функция работает со сбоями или ограничениями |
| **Примеры** | Медленная загрузка отчётов, ошибки экспорта |
| **Реакция** | В рабочее время, 2 часа |
| **Эскалация** | Лид разработки |

### И.4.3. Примеры инцидентов по уровням

| Уровень | Ситуация |
|---------|----------|
| P1 | БД недоступна > 5 минут |
| P1 | Все пользователи не могут войти |
| P1 | Данные повреждены или утрачены |
| P2 | Группа пользователей (отдел) не может работать |
| P2 | Ошибка сохранения критических данных |
| P2 | Импорт данных из ШДК не работает |
| P3 | Ошибка в отчёте (неправильные данные) |
| P3 | Медленная работа поиска |
| P4 | Опечатка в интерфейсе |
| P4 | Некритичная ошибка при экспорте |
| P5 | Визуальный баг без влияния на функцию |
