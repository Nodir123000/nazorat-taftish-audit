## З.1. ОБЩИЕ ПОЛОЖЕНИЯ

### З.1.1. Назначение

Настоящее приложение устанавливает полную спецификацию нефункциональных требований (NFR) к АИС КРУ МО РУЗ, включая требования к производительности, масштабируемости, доступности, безопасности и мобильности.

### З.1.2. Структура NFR

| Категория | Основные требования |
|-----------|-------------------|
| **Performance** | Время отклика, пропускная способность |
| **Scalability** | Рост нагрузки, объём данных |
| **Availability** | Uptime, RTO/RPO |
| **Security** | Защищённость, классы |
| **Portability** | Браузеры, ОС |
| **Accessibility** | WCAG 2.1 |

## З.10. АВТОНОМНЫЙ РЕЖИМ И ОФФЛАЙН-СИНХРОНИЗАЦИЯ (PWA)

Для обеспечения работы инспекторов в условиях отсутствия связи с центральным сервером (выездные проверки на защищенных объектах):

1. **Технологический стек:** Использование технологии Service Workers для кэширования статических ресурсов и IndexedDB для локального хранения транзакционных данных (черновиков актов, логов).
2. **Протоколы синхронизации:** При восстановлении связи (подключении к ведомственной сети) система должна автоматически инициировать фоновую синхронизацию накопленных данных.
3. **Разрешение конфликтов:** При обнаружении конфликтов версий данных (одновременное изменение одного объекта разными инспекторами) применяются следующие алгоритмы:
    * **Last Write Wins (LWW):** Для некритичных данных (описательные поля, комментарии).
    * **Manual Resolution:** Для критических данных (суммы нарушений, статусы актов) система обязана предоставить инспектору интерфейс для ручного выбора корректной версии.
4. **Контроль целостности:** Каждая запись, синхронизируемая из оффлайна, должна содержать локальную временную метку и хэш-сумму проверяющего.
