## В.1. ОБЩИЕ ПОЛОЖЕНИЯ

### В.1.1. Назначение документа

Настоящее приложение устанавливает измеримые критерии оценки достижения целей создания АИС КРУ МО РУЗ в соответствии с разделом 2 технического задания. Документ определяет целевые показатели, методологию измерения, пороговые значения и формуляры оценки.

### В.1.2. Структура критериев

Каждый критерий оценки включает:
- **Наименование показателя**
- **Целевое значение (SMART)**
- **Метод измерения**
- **Источник данных**
- **Частота измерения**
- **Пороговое значение (pass/fail)**
- **Формуляр оценки**

## В.2. ТЕХНИКО-ЭКОНОМИЧЕСКИЕ ПОКАЗАТЕЛИ

### В.2.1. Показатель 1: Сокращение временных затрат на формирование отчётности

| Атрибут | Значение |
|----------|----------|
| **Наименование** | Сокращение времени на подготовку отчётности |
| **Целевое значение** | Снижение на **≥ 60%** |
| **Единица измерения** | % (процент) |
| **Базовое значение** | 100% (текущий метод) |
| **Целевое значение** | ≤ 40% от базового |

#### Методология измерения

1. **Этап 1: Измерение базового времени (до внедрения)**
   - Зафиксировать время на подготовку каждого вида отчёта вручную
   - Виды отчётов: промежуточная справка, итоговый акт, сводная ведомость
   - Количество измерений: не менее 10 для каждого вида
   - Расчёт среднего значения: T_баз = Σ(T_i) / n

2. **Этап 2: Измерение времени после внедрения**
   - Зафиксировать время на подготовку аналогичных отчётов в АИС
   - Количество измерений: не менее 10 для каждого вида
   - Расчёт среднего значения: T_сист = Σ(T_i) / n

3. **Расчёт показателя**
   ```
   Экономия (%) = (T_баз - T_сист) / T_баз × 100%
   ```

#### Источник данных

| Источник | Тип данных | Ответственный |
|----------|-----------|--------------|
| Журнал учёта времени (до) | Excel, хронометраж | Руководитель подразделения |
| Журнал учёта времени (после) | Система учёта времени | Руководитель подразделения |
| Логи АИС | Автоматическая фиксация | Администратор системы |

#### Частота измерения

- **Первичное:** через 3 месяца после ввода в эксплуатацию
- **Повторные:** ежеквартально в течение первого года
- **Итоговое:** через 12 месяцев

#### Пороговые значения

| Уровень | Значение | Оценка |
|---------|---------|--------|
| **Отлично** | ≥ 70% | Цель превышена |
| **Хорошо** | 60–69% | Цель достигнута |
| **Удовлетворительно** | 50–59% | Цель частично достигнута |
| **Неудовлетворительно** | < 50% | Цель не достигнута |

### В.2.2. Показатель 2: Увеличение охвата объектов дистанционным контролем

| Атрибут | Значение |
|----------|----------|
| **Наименование** | Увеличение охвата дистанционным контролем |
| **Целевое значение** | Сокращение физических выездов на ≥ 50% |
| **Единица измерения** | % (процент) или количество выездов |
| **Базовое значение** | Количество выездов в год (до внедрения) |

#### Методология измерения

1. **Подсчёт выездов (до):**
   - Выгрузить данные из журнала выездов за 12 месяцев
   - Категоризировать: плановые/внеплановые, по типам проверок

2. **Подсчёт выездов (после):**
   - Выгрузить данные из АИС (модуль «Ревизии»)
   - Отфильтровать: отмеченные как «дистанционная проверка»

3. **Расчёт показателя:**
   ```
   Сокращение выездов (%) = (Выезды_до - Выезды_после) / Выезды_до × 100%
   Охват дистанционным контролем (%) = Проверки_дистанционные / Всего_проверок × 100%
   ```

#### Источник данных

| Источник | Данные |
|----------|--------|
| Журнал выездов (до) | Бумажный/Excel |
| АИС КРУ (после) | Таблица `audits`, поле `inspection_type` |

#### Пороговые значения

| Уровень | Сокращение выездов | Охват дистанционным |
|---------|-------------------|---------------------|
| **Отлично** | ≥ 60% | ≥ 70% |
| **Хорошо** | 50–59% | 60–69% |
| **Удовлетворительно** | 40–49% | 50–59% |
| **Неудовлетворительно** | < 40% | < 50% |

### В.2.3. Показатель 3: Глубина проверки данных (Data Coverage)

| Атрибут | Значение |
|----------|----------|
| **Наименование** | Глубина проверки начислений денежного довольствия и зарплат |
| **Целевое значение** | **100%** сплошная проверка |
| **Единица измерения** | % (процент охвата) |
| **Базовое значение** | Выборочная проверка (примерно 15–20%) |

#### Методология измерения

1. **Определение генеральной совокупности:**
   - Общее количество записей начислений за период
   - Источник: расчётные ведомости, интеграция с финансовыми системами

2. **Подсчёт проверенных записей:**
   - Через «шахматку» — автоматическая сверка
   - Источник: таблица `audit_verifications` в АИС

3. **Расчёт показателя:**
   ```
   Data Coverage (%) = Проверенные_записи / Всего_записей × 100%
   ```

#### Пороговые значения

| Уровень | Охват |
|---------|-------|
| **Отлично** | 100% |
| **Хорошо** | 95–99% |
| **Удовлетворительно** | 80–94% |
| **Неудовлетворительно** | < 80% |

### В.2.4. Показатель 4: Показатель комплаенс-контроля (прозрачность назначений)

| Атрибут | Значение |
|----------|----------|
| **Наименование** | Прозрачность назначения ревизионных групп |
| **Целевое значение** | **100%** назначений без конфликта интересов |
| **Единица измерения** | % (процент) |
| **Базовое значение** | Ручной контроль, формализованность ~70% |

#### Методология измерения

1. **Автоматическая проверка конфликтов:**
   - АИС проверяет каждого инспектора на конфликт интересов
   - Система блокирует назначение при обнаружении конфликта

2. **Подсчёт:**
   - Общее количество назначений
   - Назначения с выявленным конфликтом
   - Назначения, прошедшие автоматическую проверку

3. **Расчёт показателя:**
   ```
   Комплаенс (%) = (Назначения_без_конфликта / Всего_назначений) × 100%
   Покрытие_автопроверкой (%) = (Проверенные_автоматически / Всего_назначений) × 100%
   ```

#### Источник данных

| Данные | Источник |
|--------|---------|
| Назначения | Таблица `commission_members` |
| Проверки конфликтов | Таблица `conflict_checks` |
| Отклонённые назначения | Журнал `audit_log` |

#### Пороговые значения

| Уровень | Прозрачность |
|---------|-------------|
| **Отлично** | 100% |
| **Хорошо** | 98–99% |
| **Удовлетворительно** | 95–97% |
| **Неудовлетворительно** | < 95% |

### В.2.5. Показатель 5: Доступность системы (Uptime)

| Атрибут | Значение |
|----------|----------|
| **Наименование** | Коэффициент технической готовности системы |
| **Целевое значение** | **≥ 99%** (Kг ≥ 0.99) |
| **Единица измерения** | % (процент) или коэффициент |
| **Режим** | Рабочее время (08:00–18:00, Пн–Сб) |

#### Методология измерения

1. **Фиксация времени:**
   - Начало периода: T_нач
   - Конец периода: T_кон
   - Общее время: T_общ = T_кон - T_нач

2. **Фиксация простоев:**
   - Время простоя: T_простой
   - Причины: плановые/внеплановые

3. **Расчёт показателя:**
   ```
   Kг = (T_общ - T_простой) / T_общ
   Uptime (%) = Kг × 100%
   ```

#### Время простоя (допустимое)

| Тип простоя | Допустимое время (в месяц) |
|-------------|---------------------------|
| Плановое ТО | 4 часа |
| Внеплановый сбой | 30 минут |
| **Итого** | **4,5 часа** (что даёт ~99% uptime) |

#### Пороговые значения

| Уровень | Uptime | Kг |
|---------|--------|-----|
| **Отлично** | ≥ 99.5% | ≥ 0.995 |
| **Хорошо** | 99–99.4% | 0.99–0.994 |
| **Удовлетворительно** | 98–98.9% | 0.98–0.989 |
| **Неудовлетворительно** | < 98% | < 0.98 |

## В.6. МЕТОДОЛОГИЯ ОЦЕНКИ

### В.6.1. Шкала оценки

| Балл | Уровень | Описание |
|------|---------|---------|
| **5** | Отлично | Цель превышена (>100% от цели) |
| **4** | Хорошо | Цель достигнута полностью (100% от цели) |
| **3** | Удовлетворительно | Цель достигнута частично (80-99%) |
| **2** | Неудовлетворительно | Цель достигнута менее чем на 80% |
| **1** | Плохо | Цель не достигнута (<50%) |

### В.6.2. Процедура оценки

1. **Подготовка (за 2 недели до оценки):**
   - Сбор данных из всех источников
   - Заполнение формуляров измерений
   - Проверка корректности данных

2. **Проведение оценки (в течение 1 дня):**
   - Заседание комиссии
   - Представление результатов
   - Обсуждение и согласование

3. **Оформление результатов (в течение 3 дней):**
   - Заполнение итогового формуляра
   - Подписание членами комиссии
   - Утверждение руководителем

### В.6.3. Порядок оспаривания результатов

1. Направить мотивированную претензию в течение 5 рабочих дней
2. Создать согласительную комиссию
3. Провести повторное измерение при необходимости
4. Утвердить окончательные результаты
