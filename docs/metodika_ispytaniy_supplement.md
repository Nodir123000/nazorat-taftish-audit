# Приложение № 3 к Методике предварительных испытаний

## ДОПОЛНЕНИЕ: ПРОГРАММА И СЦЕНАРИИ ФУНКЦИОНАЛЬНОГО ТЕСТИРОВАНИЯ ЭЛЕМЕНТОВ ИНТЕРФЕЙСА И БИЗНЕС-ЛОГИКИ АИС КРУ МО РУз

Настоящее дополнение разработано на основе глубокого анализа программного кода и текущей реализации системы. Оно детализирует процедуры проверки ключевых функциональных модулей, описанных в п. 4.1 Методики.

### 1. Модуль «Планирование и Реестр приказов» (Planning & Orders)

**Объект испытаний:** Компоненты `UniversalOrdersRegistry` и `UnifiedOrdersView`.

| № сценария | Проверяемая функция | Ожидаемый результат |
| :--- | :--- | :--- |
| 1.1 | Генерация печатной формы приказа на ревизию | Формирование документа согласно шаблону (п. 303 `translations.ts`) с корректной подстановкой ФИО руководителя и дат выполнения. |
| 1.2 | Фильтрация приказов по статусу («Активные», «Завершенные») | Таблица отображает только записи с соответствующим флагом `status` в базе данных. |
| 1.3 | Автоматическое создание предписания на основе приказа | Перенос метаданных (№ приказа, воинская часть) в форму создания предписания без потери данных. |

### 2. Модуль «Проведение ревизии» (Audits & Execution)

**Объект испытаний:** Компоненты `FinancialActivityContent` и `FinancialAuditRegistry`.

| № сценария | Проверяемая функция | Ожидаемый результат |
| :--- | :--- | :--- |
| 2.1 | Регистрация финансового нарушения в акте | Сумма нарушения корректно суммируется в поле «Общая сумма» карточки аудита. |
| 2.2 | Передача материалов в правоохранительные органы | Перенос записи из реестра ФХД в реестр `law_enforcement` с изменением статуса на «Эскалировано». |
| 2.3 | Контроль сроков проведения (Просрочка) | Визуальное выделение красным цветом (п. 18 `universal-registry.tsx`) записей, дата окончания которых меньше текущей. |

### 3. Модуль «Кадры и KPI» (Personnel & KPI Management)

**Объект испытаний:** Компоненты `Personnel` и `KPIAnalytics`.

| № сценария | Проверяемая функция | Ожидаемый результат |
| :--- | :--- | :--- |
| 3.1 | Расчет итогового балла KPI инспектора | Формула `(Bx/Bt) * весовой коэффициент` работает корректно согласно настройкам в `kpi.management`. |
| 3.2 | Привязка инспектора к воинской части | Корректное отображение дислокации подразделения в карточке инспектора. |
| 3.3 | Автозаполнение звания при выборе ФЛ | Система подтягивает актуальное звание из связанного справочника `ref_ranks`. |

### 4. Контроль точности терминологии и локализации (GUI Verification)

В рамках испытаний необходимо подтвердить реализацию следующих уточнений в интерфейсе (на основе `lib/i18n/translations.ts`):

1. **Профессиональная терминология**: Проверка замены технических заголовков на официальные (см. Документ со спецификацией улучшений текста).
2. **Двуязычность**: Переключение между языками (RU/UZ/UZK) на любой странице должно сохранять контекст данных.
3. **Единообразие**: Один и тот же статус (например, «В процессе») должен называться одинаково во всех реестрах.

### 5. Рекомендации по оценке результатов (Checklist)

* [ ] Скорость загрузки тяжелых реестров (более 100 записей).
* [ ] Точность математических расчетов в реестре недостач (остаток = выявлено - возмещено).
* [ ] Работоспособность механизма ЭЦП (если применимо к фазе испытаний).
* [ ] Корректность отображения дислокаций на карте (если используется визуализация).
