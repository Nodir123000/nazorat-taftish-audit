# Инструкция по переносу системы "Nazorat Taftish" на другой компьютер

Эта инструкция поможет вам перенести исходный код и базу данных системы на новый компьютер.

---

## Часть 1: На старом компьютере (откуда переносим)

### 1. Архивация проекта

1. Откройте папку с проектом: `d:\Nazorat taftish`.
2. **ВАЖНО:** Не нужно копировать огромную папку `node_modules`. Она весит много и будет создана заново на новом ПК.
3. Скопируйте следующие файлы и папки в отдельное место (например, на флешку):
   - Папка `app`
   - Папка `components`
   - Папка `lib`
   - Папка `prisma`
   - Папка `public`
   - Папка `scripts`
   - Папка `styles` (если есть)
   - Папка `docs`
   - Файл `.env` (очень важно! там настройки подключений)
   - Файл `package.json`
   - Файл `tsconfig.json`
   - Файл `next.config.mjs` (или .js)
   - Файл `postcss.config.js`
   - Файл `tailwind.config.ts` (или .js)
   - Другие конфигурационные файлы в корне (eslintrc, gitignore и т.д.)

   *Или просто скопируйте всю папку, но удалите из копии `node_modules` и `.next` перед переносом, чтобы сэкономить место.*

### 2. Резервное копирование Базы Данных (PostgreSQL)

Вам нужно сохранить данные из базы `nazorat_taftish`.

#### Способ А: Через pgAdmin (графический интерфейс)

1. Откройте **pgAdmin 4**.
2. Найдите базу данных `nazorat_taftish` в дереве слева.
3. Нажмите на неё правой кнопкой мыши -> **Backup...** (Резервная копия).
4. В поле "Filename" укажите путь, куда сохранить файл (например, на флешку: `D:\backup_nazorat.sql`).
5. Вкладка "Dump options": убедитесь, что включены данные и схема.
6. Нажмите **Backup**.

#### Способ Б: Через командную строку (если установлен PostgreSQL)

Если вы знаете, где лежит `pg_dump.exe` (обычно `C:\Program Files\PostgreSQL\16\bin\pg_dump.exe`):

1. Откройте командную строку (cmd).
2. Выполните команду (замените путь на свой):

   ```cmd
   "C:\Program Files\PostgreSQL\16\bin\pg_dump.exe" -U postgres -h localhost -d nazorat_taftish -f "D:\backup_nazorat.sql"
   ```

   (потребуется ввести пароль БД: `muslima-3001##`).

## Часть 2: На новом компьютере (куда переносим)

### 1. Установка программ

1. **Node.js**: Скачайте и установите версию LTS с сайта [nodejs.org](https://nodejs.org/).
2. **PostgreSQL**: Скачайте и установите PostgreSQL с [postgresql.org](https://www.postgresql.org/download/windows/).
   - В процессе установки запомните **пароль** суперпользователя `postgres`.

### 2. Настройка Базы Данных

1. Откройте **pgAdmin 4** (установится вместе с Postgres).
2. Создайте базу данных:
   - Правый клик на "Databases" -> Create -> Database.
   - Имя: `nazorat_taftish`.
3. (Опционально) Если вы хотите использовать тот же пароль, что в старой системе, измените пароль пользователя `postgres` на `muslima-3001##`.
   - Правый клик на "Login/Group Roles" -> postgres -> Properties -> Definition -> Password.

### 3. Развертывание проекта

1. Скопируйте папку проекта с флешки на диск (например, в `D:\Nazorat taftish`).
2. Откройте командную строку (или терминал VS Code) в этой папке.
3. Установите зависимости:

   ```cmd
   npm install
   ```

   (Это займет время, так как скачиваются библиотеки).

### 4. Настройка подключения

1. Откройте файл `.env`.
2. Проверьте строку `DATABASE_URL`.

   ```env
   DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/nazorat_taftish
   ```

   Если вы указали другой пароль при установке Postgres, замените `muslima-3001##` на ваш новый пароль.

### 5. Восстановление Базы Данных

#### Способ А: Через pgAdmin

1. В pgAdmin выберите базу `nazorat_taftish`.
2. Правый клик -> **Restore...** (Восстановление).
3. В поле "Filename" выберите ваш файл `backup_nazorat.sql` с флешки.
4. Нажмите **Restore**.

**Возможные проблемы:** Если база не пустая, возникнут ошибки. Лучше восстанавливать в новую пустую базу.

#### Способ Б: Через Prisma (если бэкап не сработал или нужна чистая структура)

Если у вас нет бэкапа данных, но вы хотите просто запустить систему с нуля:

```cmd
npx prisma db push
npx prisma db seed
```

*Внимание: это создаст пустую структуру и заполнит её тестовыми данными (если настроен seed), но старые данные будут потеряны.*

### 6. Генерация клиента Prisma

После восстановления БД выполните:

```cmd
npx prisma generate
```

### 7. Запуск

```cmd
npm run dev
```

Откройте браузер по адресу `http://localhost:3000`.

---

## Решение проблем

- **Ошибка подключения к БД:** Проверьте, запущен ли службы PostgreSQL и правильный ли пароль в `.env`.
- **Ошибки зависимостей:** Удалите папку `node_modules` и файл `package-lock.json`, затем снова запустите `npm install`.
- **Ошибки Prisma:** Выполните `npx prisma generate`.
